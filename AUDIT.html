Ferry · Architecture Audit

How the agents, tools & routing actually work — and what to change

A hard look at Ferry's cloud-burst pipeline for the Cerebras × Gemma 4 hackathon: routing, single-agent tool calls, the orchestrator-workers fan-out, code execution, and web search. Every claim below was read from the code or confirmed with live Cerebras calls.

TL;DR

The core loop is genuinely good and the demo story is strong. Three things will make it noticeably better for judges, and one tool bug quietly caps quality.

P0

Role-based reasoning

Turn Gemma 4's reasoning low on for the BRAIN/EYES (plan, judge, search) and none for HANDS/VOICE (tool calls, final answer). It's a visible Gemma feature and it's free to show off.

P0

Reuse the E2B sandbox

Today every run_code spins up and kills a fresh sandbox — no state survives between steps and pip install repeats each call. Reuse one per agent run.

P0

Stop over-routing to cloud

HARD_KEYWORDS sends "translate", "exactly", "story", "code" to the cloud. That undercuts the local-first pitch and burns the 100k tok/min budget.

Live finding · Gemma 4 reasoning on Cerebras tested just now

Confirmed against the configured model gemma-4-31b from the repo .venv. Reasoning works — and it consumes output tokens, which is the catch.

The footgun: with reasoning on, a small max_tokens is spent entirely on the hidden reasoning trace and the user gets an empty bubble (finish_reason: length). This is the exact same failure mode we already hit locally with gemma4:e2b at LOCAL_MAX_TOKENS=64. Rule: reasoning ⇒ generous max_tokens.

Today Ferry uses a single global CEREBRAS_REASONING_EFFORT=none for every cloud call (clients.py · _cerebras_payload). That's safe but leaves Gemma's best party trick on the table. See Finding 2.

Findings & proposed changes

F1

Routing over-commits to the cloud

P0
Now
router.py · HARD_KEYWORDS has ~55 triggers including translate, code, generate, story, poem, exactly, summarize, repeat, today. Any hit ⇒ cloud. "Reply with exactly…" and "translate good morning" both burst to Gemma 4.
Issue
Undercuts the headline pitch ("local-first; only hard prompts leave the device"), adds latency to easy asks, and spends the 100k tok/min budget on things Liquid/e2b handle fine. It also makes the local model look unused in a demo.
Change
Split the list by intent: a small needs-tools set (search, web, latest, news, csv, pptx, png, run code, file…) that genuinely needs the cloud, vs a maybe-hard set that should defer to length / the LLM router. Drop pure-language triggers (translate, story, poem, summarize, exactly, repeat) — let those stay local.
Why
Restores the local-first story judges can see (instant answers offline), and protects the token budget that the multi-agent demo leans on.
F2

One global reasoning setting wastes Gemma 4's edge

P0
Now
_cerebras_payload applies one CEREBRAS_REASONING_EFFORT (default none) to every cloud call: planner, workers, synthesis, and the final answer all get the same setting.
Issue
If left none, the orchestrator never "thinks" about decomposition or synthesis. If flipped high globally, the tool-calling loops waste tokens reasoning about function calls and risk the empty-bubble footgun.
Change
Pass reasoning_effort per role (EYES/BRAIN low, HANDS/VOICE none) via a small param on _cerebras_payload. Pair every reasoning-on call with a generous max_tokens floor (≥ 512) so the answer survives.
Why
Better plans and synthesis where it matters, fast cheap tool loops where it doesn't — and it makes a Gemma-specific capability visible to judges (bonus: stream the reasoning as a collapsible "thinking" trace, see F7).
F3

E2B sandbox is created & killed on every run_code call

P0
Now
tools.py · _run_code_sync does Sandbox.create(...) at the top and sbx.kill() in finally — a brand-new micro-VM per call, torn down immediately.
Issue
Three costs: (1) no persistence — a file or variable made in step 1 is gone by step 3, so multi-step build-then-use flows silently fail; (2) pip install python-pptx/moviepy repeats every call (slow, flaky); (3) a cold start per call blunts the "Cerebras is instant" story right at the action step.
Change
Create one sandbox lazily per agent run, thread it through the tool loop, and kill it once when the run ends. Keep the artifact-diffing as-is. Optionally warm it at run start so the first run_code is already hot.
Why
Real multi-step coding (build a dataset → chart it → assemble a PPTX), seconds shaved off every artifact, and far fewer flaky reinstalls — directly lifts the most impressive demo (file generation).
F4

Forced tool_choice is a blunt instrument

P1
Now
cerebras_agent sets force_run_code when an artifact word appears and force_web_search when both search + artifact appear, forcing those functions before letting the model choose.
Issue
It's there because Gemma sometimes narrates code instead of calling the tool — a real problem. But forcing can fire run_code when it isn't needed, or search when the answer is already known, adding latency.
Change
Keep the forcing as a first-step nudge only (already mostly true), and lean on a tighter system prompt ("never narrate code — call run_code"). Add a one-shot "you forgot to call the tool" reprompt instead of hard-forcing every step.
Why
Keeps the reliability win without paying for unnecessary tool calls on simple prompts.
F5

Multi-agent fan-out vs a single key's token budget

P1
Now
Workers launch as unbounded concurrent tasks (asyncio.create_task per agent), each a multi-step tool loop. One key = 100k tok/min, 100 req/min. This session's trimming + 429 backoff already pulled it back from the brink.
Change
(a) Bound workers with a small asyncio.Semaphore (e.g. 3). (b) Pool multiple keys — CEREBRAS_API_KEYS is already round-robin; adding teammates' keys multiplies the per-minute budget linearly. (c) Cap MULTIVERSE_AGENTS at 3 for the demo.
Why
Makes the flashiest path — parallel specialists answering in seconds — reliable on stage, and turns Cerebras's throughput into the visible win.
F6

Web search has no recency bias for "latest/today"

P2
Now
web_search (Exa) uses type:"auto", 500-char text + highlights, snippet capped to 600 — well-bounded and safe.
Change
When the query contains latest/today/current/news, pass Exa a startPublishedDate (last ~30 days) and prefer type:"neural". Optionally include the publish date in each result line so the synthesis can cite freshness.
Why
"Current events" is a core reason to leave the device; making it visibly fresh sells the EYES capability.
F7

Reasoning & tool steps aren't surfaced as a "thinking" trace

P1
Now
Tool steps stream as italic status lines (🔍 searching…, 🐍 running code) — good. The Gemma reasoning field is dropped entirely.
Change
For BRAIN/EYES calls, stream the reasoning into a collapsible <details>Thinking…</details> block (Open WebUI renders markdown details). Keep VOICE clean.
Why
Shows judges, live, that this is Gemma 4 reasoning on Cerebras — the exact thing the prize rewards — without cluttering the final answer.
F8

Routing keyword lists overlap and are hard to maintain

P2
Now
Three overlapping tuples — HARD_KEYWORDS, MULTI_AGENT_KEYWORDS, SINGLE_AGENT_KEYWORDS — plus a local-LLM probe on borderline cases.
Change
Collapse to one table mapping keyword → capability (tool, research, language) and derive all three decisions from it. Document the precedence in one place.
Why
Easier to explain in the write-up and less likely to drift — clarity is a judging axis too.

What's already solid — don't break it

Held-open SSE delivery

Placeholder → heartbeats → tokens stream into the same bubble, no Open WebUI plugin. This is the clever core; keep the contract intact.

Single ferry model + internal routing

model="ferry" is hardcoded server-side, so stale client model ids never break. No user toggle — Ferry decides.

Orchestrator-workers (this session)

Workers get only self-contained instructions; the orchestrator plans, fixes, and synthesizes. Token-light and clean.

Artifact safety

Path-traversal guard (_artifact_target), size/count caps, dotfile filtering. Don't loosen these when adding sandbox reuse.

Resilience (this session)

History trimmed at the cloud boundary, 429 backoff-retry, real Cerebras error bodies surfaced instead of a bare "400".

Queue/return notifications (this session)

macOS locally, ntfy push to phone from the headless Jetson — nails the "walk away, get pinged" story.

Through the judges' lens

Use of Cerebras

Speed is the weapon. Lead the demo with parallel workers answering near-instantly (F5), and shave the E2B cold start (F3) so the action step feels as fast as inference.

Use of Gemma 4

Make reasoning visible (F2 + F7) and lean on tool-calling + the local Gemma/Liquid edge model. The prize rewards showing Gemma doing something only Gemma does here.

Technical execution

Offline queue + burst + held-open SSE is already a differentiator. The resilience fixes mean it won't faceplant on a "deep research" prompt mid-demo.

Story & clarity

"Local-first AI for intermittent connectivity, bursting to Gemma 4 on Cerebras." Tightening routing (F1) and keeping the EYES/BRAIN/HANDS/VOICE framing makes the architecture explainable in one breath.
